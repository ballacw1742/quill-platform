# Phase G — Drawing-Driven Estimate + Schedule (Cost & Schedule Package)

**Goal:** Charles uploads design drawings (PDF, IFC, DWG/DXF, RVT), and the agent fleet produces:
1. An AACE-classified estimate (Class 5 / 4 / 3 / 2) with cost-code breakdown
2. A schedule at the level the design supports (Level 5 detailed → Level 1 summary)
3. A "basis of estimate" + "basis of schedule" narrative
4. An explicit list of what design info would unlock the next class up
5. A risk register sized to the class

Output is a `cost_schedule_package` artifact that lands in the Approval Queue first, then publishes to Documents on approval.

## AACE classification (anchor reference)

Per AACE 18R-97 / 56R-08:

| Class | Design maturity | Accuracy range | Use |
|---|---|---|---|
| **Class 5** | 0-2% complete | -50% / +100% | Concept screening, order-of-magnitude |
| **Class 4** | 1-15% | -30% / +50% | Study/feasibility |
| **Class 3** | 10-40% | -20% / +30% | Budget authorization, control |
| **Class 2** | 30-70% | -15% / +20% | Control budget, bid baseline |
| Class 1 | 65-100% | -10% / +15% | (Not generated by agents — full bid docs required) |

The classifier reads the design set and decides which class is supportable.

## Architecture

```
[User uploads drawings: PDF / IFC / DWG / DXF / RVT]
        │
        ▼
[POST /v1/estimates/upload — multipart, returns upload_id]
        │
        ▼
[Drawing Extraction Pipeline]
   ├─ PDF → text + images (with vision-readable page renders)
   ├─ IFC → IfcOpenShell → structured BIM (entities, quantities, properties)
   ├─ DWG/DXF → ezdxf → entities, blocks, layers, dimensions
   └─ RVT → Forge/APS API (or rvt2obj fallback) → metadata + extract
        │
        ▼
[design-classifier agent (Sonnet 4.6 + vision)]
   - Reads extracted manifest + sample renders
   - Outputs AACEClassification: class + confidence + supporting_evidence + missing_for_next_class
        │
        ▼
[Approval Queue: "Class N classification — confirm before estimating?"]
   - Lane 2 (Charles signs)
   - On approve, dispatch estimator-scheduler chain
        │
        ▼
[estimator-scheduler agent (Opus 4.7 with extended thinking)]
   - Reads classification + extracted scope + cost library + project context
   - Outputs CostSchedulePackage: estimate (cost-code rows), schedule (activities + relationships),
     basis-of-estimate, basis-of-schedule, risk register, contingency, missing-info-to-next-class
        │
        ▼
[Approval Queue: "Cost & Schedule Package — Class N — review"]
   - Lane 3 (Charles + partner) for Class 3+ since money/schedule commitments increase
        │
        ▼
[On approve → publish_artifact → Documents tab + Drive export]
        │
        ▼
[User can: view in Documents, download estimate CSV, export Gantt PNG, export P6 XER]
```

## API

### Endpoints

```
POST /v1/estimates/upload
  multipart/form-data: files[], project_label (str), notes (str)
  → { upload_id, file_count, total_bytes, extraction_started: bool }

GET /v1/estimates/{upload_id}/status
  → { status: queued|extracting|classifying|estimating|done|failed,
      classification: AACEClassification | null,
      package: CostSchedulePackage | null,
      errors: [] }

POST /v1/estimates/{upload_id}/start_estimation
  Called after AACE classification is approved. Triggers the estimator-scheduler agent.
  → { approval_id }

GET /v1/estimates/{upload_id}/export?format=md|csv|xer|pdf
  → file download (xer only available when schedule.level >= 3)
```

### Storage

- Uploaded files → MinIO: `estimates/<upload_id>/raw/<filename>`
- Extraction artifacts → MinIO: `estimates/<upload_id>/extracted/{summary.json,renders/page-N.png,ifc-entities.json}`
- Classification + Package → published as Documents on approve

## Drawing extraction pipeline

New service module `api/app/services/drawings.py`:

| Format | Library | What we extract |
|---|---|---|
| **PDF** | `pdfplumber` + `pypdfium2` for renders | Title block text, scale, sheet number, dimensions/callouts via OCR (Tesseract), page renders for vision |
| **IFC** | `ifcopenshell` | Entities (walls, doors, equipment), spaces, quantity takeoffs (IfcQuantityArea, IfcQuantityVolume), property sets |
| **DWG/DXF** | DWG → DXF via `LibreDWG` (or ODA File Converter if licensed) → `ezdxf` | Entities by layer, blocks, dimensions, hatches, paper-space layouts |
| **RVT** | Forge/APS Model Derivative API (cloud) OR `rvt2obj` (local fallback) | Element categories, parameters, quantities — best when we have APS access |

Phase G.1 ships **PDF + IFC + DXF**. RVT and DWG-via-LibreDWG ship in G.4 (the polish phase) since they have moving parts that can fail in a dev environment without APS credentials.

For DWG specifically, document a manual fallback: user converts to DXF in any CAD tool first.

## UI

### Upload entry point

A new floating button **"+ Estimate from drawings"** on the `/today` tab, top-right of the hero card. Tap → opens an upload sheet:

- **Drag-drop** (desktop) or **file picker** (mobile)
- Multiple files at once
- Project label (free text, defaults to "QPB1 — <today>")
- Notes (free text)
- "Upload" primary button → triggers POST /v1/estimates/upload

### Estimate progress page

`/estimates/[upload_id]` — full-screen stack page (not a tab):

- Top: file list with extraction status badges (✓ ok / ⚠ partial / ✗ failed)
- Middle: AACE classification once available — Class N badge, confidence gauge, evidence list, missing-for-next-class list
- Bottom: "Approve classification & generate estimate" button (Lane 2 passkey re-auth)
- After estimate runs: shows summary metrics (total cost, total duration, top risks) and a "View full package" button → published doc

### Estimate-package detail view

A new variant of the document detail page (`/documents/[id]` already supports the artifact type — extends with):

- **Hero metrics:** total $, $/SF, $/MW, total duration, AACE class, schedule level
- **Estimate tab:** cost-code table (CSI-grouped, sortable, exportable to CSV)
- **Schedule tab:** Gantt rendering (use `frappe-gantt` — small dep)
- **Risk register tab:** sortable risk list
- **Basis tab:** basis-of-estimate + basis-of-schedule prose
- **Next-class tab:** missing info list with cost-to-complete

## Hard rules

1. **No commitments to owners** — every estimate output explicitly says "for internal use unless approved for external distribution." Lane 3 dual-sig required for any external-facing release.
2. **Never invent quantities or dimensions** — every quantity in the estimate cites the source (drawing ref + page/element). If the agent can't find supporting evidence, the row must be flagged with confidence < 0.5 and `rate_source: llm_estimate`.
3. **AACE class is the agent's call**, not the user's — but the user can override (Charles signs the classification approval; if he disagrees with the agent's class, he edits and re-approves).
4. **Schedule level follows class** — Class 5 → Level 1 (summary), Class 3 → Level 3 (control), Class 2 → Level 4-5 (detail).
5. **Vision-readable but never exfiltrated** — drawings stay in MinIO; only summary text + a small set of representative renders go to the LLM.
6. **No SoR writes from this flow** — published Documents are the only output. P6 XER is a manual download, not a P6 import.

## Out of scope this phase

- Real-time collaboration on the estimate (single-author)
- Per-line annotation/discussion
- Multi-currency
- Reverse-engineering an estimate to find optimal scope
- 4D BIM (schedule × geometry visualization)
